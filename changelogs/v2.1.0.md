# schema-dsl v2.1.0

Released: 2026-06-26

## Release theme

This release promotes the current main branch stabilization work to the first v2 minor line after `2.0.11`. It focuses on side-effect controlled package entries, runtime isolation, the shared `s` / `dsl` namespace, declarative extensions, array item DSL ergonomics, and the deep validation hardening completed during the review cycle.

## Added

- Side-effect controlled package entries: `schema-dsl/pure`, `schema-dsl/compat`, `schema-dsl/register-string`, `schema-dsl/string-types`, `schema-dsl/runtime`, `schema-dsl/transform`, and `schema-dsl/esbuild`.
- Runtime isolation through `createRuntime()`, including instance-level type registry, locale/messages, validator state, cache control, and lifecycle helpers.
- Shared `s` / `dsl` namespace API with equivalent callable DSL strings, object DSL compilation, factory methods, and `.require()`.
- Compile-time String DSL transform coverage plus an optional esbuild adapter.
- Declarative custom extension registration through `registerExtensions([... ] as const)` and `runtime.registerExtensions([...])`.
- Extension parameter parsing, defaults, object/positional factory arguments, `segmentMode`, and structured diagnostics.
- DSL object array item support through `s.array({ ... })` and `.items({ ... })`.

## Changed

- Babel AST packages and esbuild are optional peer dependencies for transform/esbuild consumers, keeping ordinary runtime installs smaller.
- Documentation and runnable examples now describe the side-effect controlled entry model and the three-entry custom extension model.
- Website build routing keeps the package CI path compatible with Node.js 18 while running the Rspress build on supported Node versions.

## Fixed

- Hardened raw JSON Schema classification across root helpers, runtime, conditional branches, and schema utilities.
- Completed Conditional execution coverage for nested applicators, local `$ref` chains, dependency schemas, `prefixItems`, `contains`, `propertyNames`, and public `Validator.validate()` paths.
- Aligned validation option contracts for `coerce`, `smartCoerce`, `allErrors`, sync async-custom-validator guards, and repeated conversion behavior.
- Preserved boolean schemas and dependent constraints in `SchemaUtils.pick()` / `omit()` / `partial()` workflows.
- Improved MySQL and PostgreSQL exporter reporting for supported constraints and unsupported loss detection.
- Fixed TypeScript inference gaps for unions, enum forms, numeric/boolean literals, constrained arrays, and recursive boolean JSON Schema positions.
- Improved `SchemaHelper` stable serialization for circular arrays and process-local function identity.
