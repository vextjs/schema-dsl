# v2.0.8 — Validator Cache Lifecycle Patch (2026-06-10)

> Patch release for the v2.0 latest line. This release bounds Validator and AJV compiled schema lifecycle state, closes locale cache growth paths, and makes official plugin uninstall cleanup deterministic.

---

## Runtime

- Added stable internal schema cache keys for repeated DSL definitions and `DslBuilder` materialization so repeated validation no longer creates unbounded AJV compiled schema references.
- Made Validator cache eviction and `clearCache()` release managed AJV internal schema refs instead of only clearing the outer `CacheManager`.
- Added a bounded internal compile cache for `_removeAdditional` validation paths.
- Made flattened locale message caching bounded and revision-aware so unknown locale strings and runtime locale updates do not leave stale or unbounded entries.

## Plugins

- Updated official plugin uninstall paths to clean custom formats, custom types, keywords, global plugin state, and Validator caches.
- Added `DslBuilder.unregisterType(name)` for targeted custom type cleanup while preserving `clearCustomTypes()` for full isolation resets.

## Documentation

- Documented `DslBuilder.unregisterType(name)` in the API reference, custom type registration guide, and union type guide in both English and Chinese.
- Added this release note to the package changelog and reset the unreleased changelog section.

## Tests

- Added focused Validator memory lifecycle tests for stable cache reuse, AJV ref release, bounded locale cache behavior, locale invalidation, and `_removeAdditional` cache cleanup.
- Extended official plugin entry tests to verify uninstall cleanup for formats, custom types, keywords, and cache state.
