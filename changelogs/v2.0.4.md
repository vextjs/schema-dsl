# v2.0.4 — Explicit String Extensions & v1 Compatibility Patch (2026-06-03)

> Patch release for the v2.0 latest line. This release preserves the v2.0.3 public package shape while making v1 string-extension compatibility explicit and safer for modern consumers.

---

## Fixes

- **String extensions:** root import no longer installs `String.prototype` extensions as a global side effect.
- **v1 compatibility:** `installStringExtensions()` remains available and supports the legacy no-argument call form.
- **Builder compatibility:** string extension helpers continue to route through the current `DslBuilder` behavior.
- **Tests:** v1 compatibility coverage now asserts both the no-global-side-effect root entry and explicit string-extension installation behavior.

---

## Documentation

- Updated quick-start, API reference, and string-extension documentation to describe the explicit installation path.
- Updated document version headers for the current v2.0.4 patch line.
