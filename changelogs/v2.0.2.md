# v2.0.2 — Compatibility Patch (2026-06-01)

> Patch release focused on smooth upgrades for existing v1 consumers and release metadata hygiene.

---

## Fixes

- **Compatibility (entry types):** exported `ErrorMessages` and `ErrorCodeMap` from the root entry so consumers can import the public error template types directly from `schema-dsl`.
- **Compatibility (`dsl` overloads):** restored the optional `SchemaIOOptions` second argument for `dsl(object, options?)` and `dsl(string, options?)`, keeping v1-style call sites source-compatible.
- **Compatibility (`dsl` builder typing):** narrowed the root `dsl('...')` overload to the concrete `DslBuilder` type so consumers that rely on builder-specific APIs can compile cleanly.
- **Compatibility (`validate` generics):** relaxed the top-level `validate<T>()` and `validateAsync<T>()` input parameter to `unknown` while preserving typed return data, avoiding false-positive TypeScript errors when validating untrusted input.
- **Release:** synchronized package metadata, changelog coverage, lockfile versioning, audit gate and npm repository URL normalization before tag-based publish.
