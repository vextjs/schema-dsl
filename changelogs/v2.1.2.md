# schema-dsl v2.1.2

Released: 2026-07-07

## Release theme

This patch prepares the current main branch for the next release after `2.1.1`. It closes release-readiness drift, fixes the remaining raw JSON Schema `default` classifier edge, and refreshes the maintained performance baseline from the release-candidate full benchmark.

## Fixed

- Treat annotation-only JSON Schema inputs such as `{ default: 'fallback' }` as raw JSON Schema at the root helper boundary.
- Preserve DSL object field semantics for keyword-like field names such as `{ default: 'string!' }`.
- Invalidate root fast-path metadata and Validator compile caches when caller-owned raw schemas change shape after reuse.
- Reject legacy JSON Schema `validate` keyword functions that are async or return Promise-like values in sync validation instead of silently passing them.
- Execute legacy JSON Schema `validate` keyword functions through `validateAsync()` traversal, including nested property paths.
- Update GitHub security policy metadata for the 2.1.x line and fix the issue template FAQ link.
- Add regression coverage for FastPlan fallback reasons, smart coercion metadata, public entry fast paths, official plugin examples, and AJV skipped-property applicators.

## Changed

- Updated public performance docs to the release-candidate Node.js v20.20.2 / Windows x64 full benchmark:
  - S1 simple valid object: `~1.822M ops/s`
  - S2 invalid object without i18n formatting: `~243K ops/s`
  - S3 nested valid object: `~1.067M ops/s`
- Recorded the current extended Zod scenario matrix as schema-dsl `14/19` wins and Zod `5/19` wins. This is maintained as a regression signal, not as a public marketing claim.
- Synchronized the FAQ and design philosophy pages with the current performance-guide baseline.
