# schema-dsl v2.1.1

Released: 2026-06-30

## Release theme

This patch release tightens validation memory lifecycle behavior without changing public validation semantics. It focuses on bounded matcher reuse, batch compile-cache prewarming, and more complete exporter loss reporting for local `$ref` targets.

## Fixed

- Hardened schema object construction and validation for reserved property names such as `__proto__`.
- Bounded the static `Validator.quickValidate()` AJV schema cache with explicit cleanup support.
- Added compatibility-safe JSON-only i18n directory loading through `codeLocaleFiles: 'deny'`.
- Bounded `patternProperties` `RegExp` matcher reuse in normal and conditional validation paths, and clear those matcher caches through `Validator.clearCache()`.
- Prewarmed `Validator.validateBatch()` through the managed compile cache while preserving the same per-item validation path, options behavior, conditional handling, smart coercion, and custom validator semantics as `validate()`.
- Expanded exporter loss reporting so local `$ref` targets are traversed and circular schema object graphs do not recurse indefinitely.
