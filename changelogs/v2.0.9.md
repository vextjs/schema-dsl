# v2.0.9 — Documentation and Dependency Convergence Patch (2026-06-11)

> Patch release for the v2.0 latest line. This release closes the document/example review loop, keeps the refreshed docs site design, and upgrades cache/test dependencies without changing the public runtime API.

---

## Documentation and Examples

- Localized the Chinese documentation navbar and refreshed the schema-dsl logo/theme while preserving light/dark behavior.
- Fixed home hero action button overflow and refined the technical visual treatment.
- Clarified request-time DSL usage, cache hit boundaries, high-cardinality dynamic schema risks, and server-side `Validator` lifecycle guidance.
- Added `examples:run` as a runnable documentation example gate and documented the requirement that docs and examples must stay synchronized.
- Expanded high-risk examples with runnable assertions for Validator lifecycle, DSL syntax, conditional rules, exporters, FAQ, troubleshooting, and custom keywords.
- Closed the full document review by fixing API reference coverage for all root exports, repairing emoji-heading anchors, and aligning `type-converter` example output comments.

## Runtime and Exporters

- Aligned `MarkdownExporter` and `SchemaUtils` Markdown/HTML output with label-vs-description docs by preserving display labels and field descriptions separately.
- Updated `CacheManager` to construct cache-hub `MemoryCache` with cache-hub 2.x options (`defaultTtl`, `enableStats`, `enabled`) and to destroy old cache instances after capacity rebuilds.
- Kept the public `CacheManager` API unchanged for v2 consumers.

## Dependencies

- Upgraded `cache-hub` from `1.0.0` to `2.2.4`.
- Upgraded `vitest` and `@vitest/coverage-v8` from `3.2.4` to `3.2.6` to clear the npm audit advisory while preserving the project `node >=18.0.0` baseline.
