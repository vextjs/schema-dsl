# schema-dsl v2.0.11

Released: 2026-06-15

## Fixes

- Share the custom type registry across ESM and CJS package entrypoints in the same Node.js process.
- Keep `DslBuilder.getCustomTypes()`, `clearCustomTypes()` and `TypeRegistry.setStrict()` consistent across ESM/CJS entrypoints.
- Fix downstream frameworks that compile user route files to CJS while using schema-dsl from ESM for OpenAPI or validation conversion.
- Restore v1-compatible root type exports: `CacheManagerOptions`, `ErrorMessageConfig`, `LocaleMessages`, and `Plugin`.
- Include dynamic custom types in `TypeRegistry.entries()` and make static/dynamic re-registration replace the previous registration deterministically.

## Validation

- `npm run build`
- `npm run test:interop`
- `npx vitest run test/unit/parser/TypeRegistry.test.ts`
