# schema-dsl v2.1.6

Release date: 2026-07-13

## Added

- Exported `BabelPeerDependencyError` from `schema-dsl/transform` for typed handling of missing optional Babel peers.

## Changed

- Bound the bilingual performance guide to one tracked snapshot and calibrated regression checks against the pinned same-run Zod workload.
- Required every benchmark entry to build current sources before measurement.
- Strengthened the tag-triggered publish workflow with coverage, documentation example, website build, packed-consumer, audit, and performance smoke gates.
- Aligned the active project Profile with current scripts, package positioning, and release boundaries.

## Fixed

- Invalidated generated and caller-owned Schema caches when their public structures change after warm-up.
- Preserved composition-only and dependency-only fields in `SchemaUtils.omit()` projections.
- Executed `minContains` / `maxContains` consistently in sync, async, quick, runtime, and compiled validation paths while retaining the Draft 7 baseline.
- Honored zero-match and bounded `contains` ranges while compensating AJV-skipped own-property schemas.
- Preserved the active `Validator` instance formats and options while matching skipped own-property schemas inside applicator branches.
- Evicted projected `contains` schemas and async preflight clones by their original public schema owner without weakening duplicate-`$id` detection between different schemas.
- Restored deprecated `ValidateOptions` source-compatibility fields and preserved `SchemaUtils.withPerformance()` assignability.
- Unified Schema-position traversal for `additionalItems` across Conditional execution, async custom validators, SchemaUtils, exporter loss reports, and skipped-property compensation.
- Corrected prefix/repeated item custom-validator binding, multi-item skipped-property checks, and local `$ref` root-scope handling.
- Made plugin uninstall ownership manager-local so observer unregister/clear cannot tear down another manager's installed resources.
- Split official `custom-format` leases by AJV and `DslBuilder` identity so independent Validators sharing one Builder retain DSL types until the final owner uninstalls.
- Made `PluginManager.clear()` continue cleaning independent plug-ins, emit `plugins:clear-error`, retain failed plug-ins for retry, and throw one aggregate failure after the pass.
- Recognized Conditional schemas nested under schema-valued keywords in the shared raw JSON Schema classifier, keeping root sync, root async, direct Validator, and isolated runtime behavior aligned.
- Corrected bilingual documentation and executable examples for recursive `partial()`, clone semantics, and the supported JSON Schema dialect boundary.
- Corrected the performance matrix diagnostic classification and refreshed the tracked local snapshot to schema-dsl `14/19`, Zod `5/19`.

## Compatibility

- No public package entry, exported symbol, function signature, or documented root/pure/runtime behavior was removed.
- `PluginManager.clear()` now throws an `AggregateError` when one or more uninstall hooks fail; this is an explicitly accepted lifecycle-correctness change for the current pre-adoption project state.
- `JSONSchemaInput.additionalItems` is now explicit in the public type and remains compatible with existing index-signature usage.
- The fixes tighten previously skipped validation paths; invalid data that was incorrectly accepted may now be rejected.
