# v2.0.10 — DSL Alias Compatibility Patch (2026-06-13)

> Patch release for the v2.0 latest line. This release restores legacy DSL aliases used by downstream adapters while keeping schema-dsl as the single parser and type authority.

---

## Compatibility

- Added built-in aliases for legacy and adapter-facing DSL names: `int`, `float`, `double`, `decimal`, `mixed`, `buffer` and `objectid`.
- Preserved the existing unknown-type fallback behavior while adding structured diagnostics for consumers that need to distinguish unknown DSL names from accepted aliases.
- Added `compileWithDiagnostics()` so downstream libraries can delegate DSL parsing to schema-dsl without maintaining a second allowlist.

## Parser and Adapter Support

- Propagated parser options through object DSL compilation so consumers can opt into diagnostics consistently for nested schemas.
- Kept existing `dsl(...)`, `validate(...)` and String extension contracts unchanged.

## Documentation

- Refined the documentation homepage hero spacing, alignment and footer navigation while keeping the current visual identity.

## Validation

- `npm run typecheck`
- `npm test`
- targeted parser/diagnostics tests
- CJS smoke import
