# v2.0.3 — Async Custom Validation & Docs Patch (2026-06-01)

> Patch release for the v2.0 latest line. This release keeps the v1-compatible public API surface from `v2.0.2` and fixes the remaining async custom validator/runtime documentation drift found during release review.

---

## Fixes

- **Validation:** `Validator.validateAsync()` now strips `_customValidators` before the base AJV pass, then runs Promise-returning `.custom()` validators after base validation succeeds.
- **Validation errors:** async custom validator failures now preserve nested object/array paths and surface string, boolean-false, object-error and thrown-error outcomes consistently.
- **Examples:** documentation examples now avoid non-root internal parser/compiler imports and exercise only documented public APIs.
- **Docs:** README-linked docs, Rspress navigation, current document version headers, historical changelog links and async custom validator wording are aligned with the current runtime behavior.
