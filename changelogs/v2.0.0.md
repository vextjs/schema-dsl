## Breaking Changes Fixed (v1 â†’ v2 Compatibility)

### BC-2 â€” `DslAdapter.parseObject()` returns `ObjectDslBuilder`

**Before (v1 / beta):** `parseObject({ ... })` returned a plain `JSONSchema` object.

**After (v2.0.0):** Returns an `ObjectDslBuilder` instance with chainable methods.

```typescript
// v2.0.0
const builder = DslAdapter.parseObject({ name: 'string!', age: 'number' })

// Chainable API (new)
const schema = builder
  .strict()        // disallow extra properties
  .requireAll()    // require all defined properties
  .toSchema()      // â†’ JSONSchema (raw internal schema)

// Clean JSON Schema (internal markers stripped, safe for external tools)
const jsonSchema = builder.toJsonSchema()

// dsl({...}) still returns plain JSONSchema directly (no change required for dsl() callers)
const s = dsl({ name: 'string!' })  // â†’ JSONSchema (backward-compatible)
```

**Migration:** Code that calls `DslAdapter.parseObject()` and accesses `.type` / `.properties` / `.required` directly must add `.toSchema()`:

```typescript
// Before
const schema = DslAdapter.parseObject({ ... })
console.log(schema.type)  // undefined in v2!

// After
const schema = DslAdapter.parseObject({ ... }).toSchema()
console.log(schema.type)  // 'object' âœ“
```

### BC-4 â€” `DslAdapter.typeMap` Proxy + `registerType()`

`typeMap` is now a read-only Proxy over the internal type registry (live view, not a snapshot).
`registerType(name, schema)` added as a first-class registration method.

```typescript
// Register a custom type on the adapter
DslAdapter.registerType('phone-cn', { type: 'string', pattern: '^1[3-9]\\d{9}$' })

// Read live type map
console.log(DslAdapter.typeMap['phone-cn'])  // â†’ { type: 'string', ... }
```

### BC-5 â€” `ConditionalBuilder.require(field)`

Added `require(field)` to `ConditionalBuilder` for v1-style conditional field-requirement:

```typescript
const condition = dsl.if((data) => data.type === 'business')
  .require('companyName')
  .then({ taxId: 'string!' })
```

### BC-6 â€” Async validators in sync `validate()`

**Before (v1):** Using an async `.custom()` validator and calling sync `validate()` would either
throw a JavaScript exception or silently pass.

**After (v2.0.0):** Async validators in sync `validate()` return a structured validation error:

```typescript
const schema = dsl({
  value: dsl('string!').custom(async (v) => v.length > 3),
})

const result = validator.validate(schema, { value: 'ok' })
// result.valid === false
// result.errors[0].message === 'Async validation not supported in sync validate(). Use validateAsync() instead.'

// Correct approach: use validateAsync()
const data = await validator.validateAsync(schema, { value: 'valid' })
```

---

### BC-7 â€” `string:N` single-value expands to `{minLength:N, maxLength:N}`

`string:100` now expands to `{ minLength: 100, maxLength: 100 }` in `toJsonSchema()` output,
matching v1 behavior and keeping JSON Schema valid.

Internal storage uses `exactLength: N`; `toJsonSchema()` translates it automatically.

## Migration Guide (v1 â†’ v2)

### Minimal changes required for most users

1. **`dsl({...})`** â€” no change; still returns `JSONSchema` directly.
2. **`dsl('string!')`** â€” no change; still returns `DslBuilder`.
3. **`validate()` / `Validator.validate()`** â€” no change.
4. **`Locale`, `ErrorFormatter`, `DslBuilder` chains** â€” no change.

### Changes needed if you use:

| Usage | Change |
|-------|--------|
| `DslAdapter.parseObject({...}).type` | Add `.toSchema()`: `DslAdapter.parseObject({...}).toSchema().type` |
| `DslAdapter.parseObject({...}).properties` | Add `.toSchema()` |
| `DslAdapter.typeMap` (snapshot) | Now a live Proxy â€” read-only, always current |
| Async validators in sync `validate()` | Move to `validateAsync()` for async validators |
