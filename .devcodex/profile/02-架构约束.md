## 模块边界约定

| 层 | 目录 | 职责 | 依赖方向 |
|----|------|------|---------|
| **核心层** | `src/core/` | 验证引擎、DSL 编译、缓存、多语言 | 仅依赖 ajv、cache-hub、内部 utils |
| **适配器层** | `src/adapters/` | DSL 字符串解析 + 主 `dsl()` 函数实现 | 依赖 core |
| **导出器层** | `src/exporters/` | Schema → 数据库 DDL 转换 | 依赖 utils/TypeConverter |
| **插件层** | `src/plugins/` | 插件注册/卸载 | 依赖 core |
| **工具层** | `src/utils/` | 通用工具函数，无业务逻辑 | 无内部依赖 |
| **错误层** | `src/errors/` | 错误类型定义 | 无内部依赖 |
| **配置层** | `src/config/` | 常量、正则模式集 | 无内部依赖 |
| **多语言层** | `src/locales/` | 语言包数据文件 | 无内部依赖 |

> ⚠️ 依赖方向：`core` ← `adapters/exporters` ← `src/index.ts`，禁止反向依赖。
> ⚠️ `src/utils/` 和 `src/config/` 是纯工具层，禁止引入业务逻辑。
> ⚠️ **不得修改 `dist/` 目录**，所有变更在 `src/` 中进行后运行 `npm run build`。

## 主要 API 导出

### 核心 API

```typescript
// ESM
import { dsl, validate, validateAsync, config } from 'schema-dsl';

// CJS
const { dsl, validate, validateAsync, config } = require('schema-dsl');
```

### 类与工具

```typescript
import {
  DslBuilder, Validator, ErrorFormatter, CacheManager, SchemaCompiler,
  Locale, MessageTemplate,
  ValidationError, I18nError, ErrorCodes,
  installStringExtensions, uninstallStringExtensions,
  exporters, MongoDBExporter, MySQLExporter, PostgreSQLExporter, MarkdownExporter,
  TypeConverter, SchemaHelper, SchemaUtils,
  PluginManager,
  CONSTANTS, VERSION,
} from 'schema-dsl';
```

### validate() 返回值格式

```typescript
const result = validate(schema, data, { format: true, locale: 'zh-CN', coerce: true });
// Returns: { valid: boolean, data?: any, errors?: ValidationErrorItem[] }

interface ValidationErrorItem {
  field: string;
  message: string;
  keyword: string;
  type?: string;      // e.g. "required" | "type" | ...
  expected?: string;  // e.g. "string" | "number" | ...
}

// 异步（失败自动抛出 ValidationError）
const data = await validateAsync(schema, inputData);
// ValidationError: { errors, statusCode, toJSON() }
```

## 缓存层说明

- `CacheManager.ts` 使用 `cache-hub` 的 `MemoryCache` 作为 LRU 缓存后端
- 构造：`new MemoryCache({ maxEntries: this._maxSize })`（默认 100）
- API 使用：`get` / `set` / `del`（注意是 `del` 不是 `delete`）/ `has` / `clear` / `keys` / `getStats` / `resetStats`

## 架构设计模式

| 模式 | 应用模块 |
|------|---------|
| **Builder**（链式构建） | `DslBuilder` |
| **Adapter**（适配转换） | `DslAdapter` |
| **Singleton**（单例） | `Validator`（默认导出实例） |
| **Strategy**（策略） | `ErrorFormatter`（多种格式化策略） |
| **Template Method**（模板方法） | 导出器（`Exporters`） |
| **Observer**（观察者） | `PluginManager`（EventEmitter 钩子） |
| **Decorator**（装饰器） | `StringExtensions`（String 原型扩展） |
| **Cache**（缓存） | `CacheManager`（cache-hub LRU） |
| **Chain of Responsibility** | `ConditionalBuilder`（if/elseIf/else） |

## DSL 语法约定

```
'string:3-32!'      → string 类型，长度 3~32，必填
'string:10'         → string 类型，exactLength=10（精确长度）
'email!'            → email 格式，必填
'number:18-120'     → number 类型，范围 18~120
'a|b|c'             → 枚举（a 或 b 或 c）
'enum:a,b,c'        → 枚举（逗号分隔）
'array<string>'     → string 数组
'string:3-32!'.pattern(/^[a-z]+$/).label('用户名')  → 链式扩展
```

## 架构边界与已知限制

| 约束 | 说明 |
|------|------|
| **仅支持 JSON Schema Draft 7** | 不支持 Draft 2020-12 |
| **不涉及持久化** | 导出器只生成 Schema DDL，不含 DB 驱动 |
| **不支持递归 Schema** | 无法定义自引用类型 |
| **单进程缓存** | LRU 缓存不跨进程共享，多进程需独立预热 |
| **LRU 默认上限** | 100 条，可通过 `config({ cache: { maxSize: N } })` 调整 |
