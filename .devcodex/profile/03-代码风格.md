# 03 · 代码风格

## 命名约定

| 类型 | 风格 | 示例 |
|------|------|------|
| **类名** | PascalCase | `DslBuilder`、`ValidationError`、`Locale` |
| **公开方法 / 函数** | camelCase | `validate()`、`parseString()`、`addLocale()` |
| **私有方法** | `_camelCase`（下划线前缀） | `_parseType()`、`_normalizeSchema()`、`_normalizeError()` |
| **常量** | UPPER_SNAKE_CASE | `CONSTANTS`、`ErrorCodes`、`MAX_LENGTH` |
| **源文件名** | PascalCase（核心类文件） | `DslBuilder.js`、`Validator.js`、`ErrorFormatter.js` |
| **测试文件名** | `原文件名.test.js` | `DslBuilder.test.js`、`Validator.test.js` |
| **普通变量** | camelCase | `schema`、`errorList`、`currentLocale` |
| **布尔变量** | `is` / `has` / `should` 前缀 | `isValid`、`hasError`、`shouldCoerce` |
| **集合变量** | 复数形式 | `errors[]`、`messages{}`、`schemas{}` |
| **工厂方法** | `create` / `make` / `build` | `create()`、`build()`、`make()` |

### 关键规则速查

| 规则 | 要求 |
|------|------|
| **分号** | 必须有（`"always"`） |
| **引号** | 单引号（`'`） |
| **缩进** | 2 空格 |
| **换行符** | Windows 风格（CRLF） |
| **未使用变量** | 警告（下划线前缀参数豁免） |
| **console** | 允许（`"off"`） |
| **尾部逗号** | 禁止 |

| **关键字空格** | `if (` / `else {` |

> ⚠️ 项目**无 Prettier 配置**，格式化完全由 ESLint 规则管控。
> ⚠️ 换行风格为 **Windows（CRLF）**，在 Linux/macOS 提交时注意 `.gitattributes` 设置。

## 模块化风格

### TypeScript ESM（主要方式）

```typescript
// 引入依赖
import Ajv from 'ajv';
import { MemoryCache } from 'cache-hub';
import { DslBuilder } from './DslBuilder.js';

// 导出
export { dsl, validate, validateAsync, config };
export default dsl;
```

> ⚠️ `src/` 目录下所有源码使用 **TypeScript ESM**，禁止使用 `require()` 或 `module.exports`。
> ⚠️ 构建产物 `dist/` 由 tsup 生成 ESM（`.js`）和 CJS（`.cjs`）双格式，**不手动修改**。

## JSDoc 注释风格

### 类/模块头注释（每个文件必须有）

```javascript
/**
 * DSL Builder - 统一的 Schema 构建器
 *
 * 支持链式调用扩展 DSL 功能
 *
 * @module lib/core/DslBuilder
 * @version 2.0.0
 * @author rocky <rockyshi1993@gmail.com>
 *
 * @example
 * const schema = dsl('email!');
 * const schema = dsl('email!').pattern(/custom/).label('邮箱');
 */
```

### 公开方法注释（必须有）

```javascript
/**
 * 验证数据
 *
 * @param {Object} schema - JSON Schema 对象
 * @param {*} data - 待验证数据
 * @param {Object} [options] - 可选配置
 * @param {boolean} [options.format=true] - 是否格式化错误
 * @param {string} [options.locale] - 动态指定语言
 * @param {boolean} [options.coerce] - 是否启用智能类型转换
 * @returns {{ valid: boolean, data?: any, errors?: Array }} 验证结果
 *
 * @example
 * const result = validate(schema, { email: 'test@example.com' });
 * if (result.valid) console.log('通过');
 *
 * @throws {Error} 当 schema 无效时抛出
 */
function validate(schema, data, options = {}) { }
```

### 行内注释约定

```javascript
// 关键业务逻辑说明（简短清晰）
const schema = parseString(dslString);  // 转换 DSL 字符串为 JSON Schema

// ⚠️ 警告标记（需特别注意的逻辑）
// ✅ 成功路径标记
// ❌ 失败路径标记
// 📝 TODO 项标记（需后续处理）
// 私有方法不需要 JSDoc，但复杂逻辑需要行内注释
```

> 注释使用中文，技术术语（方法名、类名、API 名）保留英文原文。

## 错误处理约定

```javascript
// 同步验证：返回结果对象，不抛出
const { valid, errors } = validate(schema, data);

// 异步验证：失败时抛出 ValidationError
try {
  const data = await validateAsync(schema, input);
} catch (err) {
  // err instanceof ValidationError
  // err.errors → 错误列表
  // err.statusCode → HTTP 状态码
  // err.toJSON() → JSON 格式
}

// 多语言错误：通过 dsl.error 抛出
dsl.error.throw('account.notFound', 'zh-CN');
dsl.error.assert(value, 'account.notFound', 'zh-CN');
```

## 测试代码风格

```typescript
// 测试文件结构（Vitest）
import { describe, it, expect, beforeEach } from 'vitest';
import { dsl, validate } from 'schema-dsl';

describe('模块名 - 功能描述', () => {
  beforeEach(() => {
    // 每个测试前重置状态
  });

  it('应该 [期望行为描述]', () => {
    // Arrange
    const schema = dsl({ email: 'email!' });

    // Act
    const result = validate(schema, { email: 'test@example.com' });

    // Assert
    expect(result.valid).toBe(true);
    expect(result.errors).toBeUndefined();
  });

  it('应该在 [条件] 时 [行为]', () => {
    // 错误路径测试
    const result = validate(schema, { email: 'invalid' });
    expect(result.valid).toBe(false);
    expect(result.errors?.length).toBeGreaterThan(0);
  });
});
```

> 测试描述语言：中文，格式为 `应该 [期望行为]` 或 `应该在 [条件] 时 [行为]`。

## 代码组织约定

### 文件内部结构顺序

1. 文件头注释（模块说明）
2. `import` 导入（先 Node 内置 → 再第三方 → 再内部模块）
3. 类型定义（`interface` / `type`）
4. 常量定义
5. 类定义（或函数定义）
6. `export`

### 类内部结构顺序

1. 私有字段（`private`）
2. `constructor`
3. 公开方法（`public`）
4. 静态方法（`static`）
5. 私有方法（`private`）

### 导出格式

```typescript
// ✅ 统一使用命名导出
export { ClassName, functionName, CONSTANT_NAME };

// ✅ 类型导出
export type { InterfaceName, TypeAlias };

// ❌ 避免 default export（index.ts 除外）
```

## 文件变更后的同步要求

修改以下类型文件时，需同步更新对应文档：

| 变更类型 | 需同步的文件 |
|---------|------------|
| 新增 / 修改公开 API | `src/types/validate.ts`（接口）、`README.md` |
| 版本号变更 | `package.json`、`src/config/constants.ts`（VERSION）、`CHANGELOG.md` |
| 新增语言包 | `src/locales/`（新增 .ts 文件）、`src/core/Locale.ts`（注册）、`README.md` |
| 新增导出器 | `src/exporters/`（新增 .ts）、`src/index.ts`（导出） |
| 修改 src/ 代码 | 运行 `npm run build` 重新生成 `dist/` |
